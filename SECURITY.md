# Security Policy

## Supported versions
- Active development: main branch and the latest released version (0.1.x). Older versions are not maintained.

## Reporting a vulnerability
- Email security issues to li.jack.services@gmail.com. Please avoid filing public issues for vulnerabilities.
- Include details that help reproduce and assess impact: affected version/commit, OS, compiler version, SQLCipher/SQLite version, configuration changes, and step-by-step reproduction. Do not share real secrets or production vaults.
- You can optionally provide a minimal proof-of-concept; avoid sending malicious binaries or payloads.

## Response process
- We aim to acknowledge reports within 3 business days and will keep you updated as we investigate.
- Once a fix is ready, we will coordinate a disclosure timeline and reference you (with permission) in release notes.
- If a vulnerability is in a dependency, we will forward details upstream and note the dependency in advisories.
